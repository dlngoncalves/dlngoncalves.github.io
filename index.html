Donnie Project

Role: Research assistant and Machine Learning Engineer

Developed a CNN-powered vision system for the Donnie robot, allowing it to detect and interpret physical Braille characters; thus enabling a tactile-to-digital interface for students to interact with their environment.

Read Published Paper Watch the robot in action

My Role & Technical Focus

Donnie was designed from the ground up to teach programming to visually impaired students. One of the projects goals was giving the robot the ability to read Braille characters. My role was to provide the robot's 'sight'.

After unsuccessful attempts with printed characters due to the requirements of the project, our team created a set of large blocks with holes plus pins, which could be used to simulate large scale Braille characters.

I then developed a vision-to-logic pipeline that used a custom-trained Convolutional Neural Network (CNN) to detect and classify these Braille characters in real-time.

To achieve this, our team engineered a unique dataset of 2,800+ labeled images, overcoming classroom challenges like erratic lighting and camera angles.

This research was published at WVC 2020 and has since been cited 13 times as a performance benchmark for assistive robotics.

  • Computer Vision Pipeline: Optimized a CNN for real-time inference on the robot's onboard hardware, achieving a 92.3% accuracy rate.
  • Dataset Architecture: Created and published a novel dataset of 2,818 labeled Braille images, filling a critical data gap in the research community.
  • Tactile-to-Digital Interface: Engineered the software bridge that translates physical Braille blocks (tactile) into executable digital code (digital).
  • Academic Impact: Authored peer-reviewed published research on CNN-based Braille detection (WVC 2020), currently cited by 13 international researchers in Springer Nature, IEEE, and other high-impact venues as a performance benchmark for assistive computer vision.

🔬 Advanced Real-Time Graphics Projects in Rendering and Data Visualization

Hybrid Real-Time Raytracer

Role: Research & Graphics Programmer

Developed a custom hybrid rendering engine in C++ and OpenGL, combining rasterization (Deferred Shading) with GLSL-based ray tracing to achieve real-time reflections and shadows on mid-range(circa 2015) hardware.

Read Paper Watch Video Code

My Role & Technical Focus

Long before hardware-accelerated ray tracing became an industry standard via APIs like DXR, I developed a custom hybrid rendering engine from scratch as my undergraduate final project to explore real-time GI techniques.

This project optimized the traditional ray tracing pipeline by utilizing Deferred Shading. I rasterized primary visibility to a G-Buffer, eliminating the first level of recursion, and passed the scene geometry to the GPU via Shader Storage Buffer Objects (SSBOs).

From there, I wrote custom iterative ray-intersection logic in GLSL fragment shaders to calculate mathematically accurate shadows and multi-bounce reflections.

  • Custom Rendering Pipeline: Architected a hybrid raster/ray-traced engine from the ground up using C++ and OpenGL, explicitly managing framebuffers and G-Buffer textures.
  • Low-Level Shader Logic: Implemented the Möller-Trumbore ray-triangle intersection algorithm entirely within GLSL fragment shaders, adapting standard recursive ray tracing into an iterative GPU-friendly format.
  • Memory & Data Bridging: Utilized SSBOs (Shader Storage Buffer Objects) to efficiently stream dynamic scene geometry, material arrays, and light data from the CPU to the GPU.
  • Performance Architecture: Demonstrated that the hybrid deferred approach yielded an 83% performance improvement over a pure ray-traced baseline on mid-range hardware of the time (Radeon R7 260x).
  • Academic Recognition: Research selected for the "Workshop G2" at the SBGames 2017 Computing Track, showcasing deep proficiency in low-level graphics APIs.

TrajVis High-Performance Real-Time Visual Analytics of Multivariate Trajectory Data

Role: Master's Researcher & Lead Developer (PUCRS)

Developed a specialized visual analytics system capable of rendering and analyzing massive multivariate trajectory datasets (24M+ points) in real-time.

Dissertation Video Code

My Role & Technical Focus

To solve the "visual clutter" and performance bottlenecks common in geospatial analysis, this project moved beyond standard CPU-bound methods.

By architecting a custom GPU-based pipeline, the tool enables analysts to interactively filter, cluster, and map complex attributes (like weather and speed) across millions of data points without losing interactive frame rates.

  • Custom GPU Pipeline: Engineered a pipeline that offloads attribute mapping and multivariate filtering to GPU shaders, maintaining interactive performance even when processing datasets exceeding 5 million active points
  • Advanced Geometry Rendering: Developed a custom line-rendering routine using geometry shaders to overcome default OpenGL limitations, ensuring seamless segment joins and eliminating visual artifacts during high-zoom interactions.
  • TRACLUS Implementation: Integrated the Partition-and-Group framework (TRACLUS) to cluster raw trajectories into representative paths, significantly reducing visual complexity and revealing latent behavioral patterns
  • Multi-Source Data Fusion: Architected a system to enrich raw GPS data with external environment variables (via DarkSky/MapBox APIs), enabling complex queries such as "Bicycle commute patterns in temperatures below 0°C"
  • Geospatial Visualization: Implemented a tile-based map system using elevation heightmaps for vertex displacement, providing a 3D topographic context for trajectory analysis.
